## The 6 Essential Commands

### 1. Plan a Task: `/ccpm:plan`

```bash
/ccpm:plan "Add user authentication" my-app
```

**What it does:**
- Creates a new task
- Plans the implementation
- Generates a checklist
- All in one command

**When to use:**
- Starting a new feature
- Have a clear idea of what to build

**Old way:** `/ccpm:planning:create` + `/ccpm:planning:plan` (2 commands)

---

### 2. Start Work: `/ccpm:work`

```bash
# In your feature branch: feature/PSN-30-auth
/ccpm:work
```

**What it does:**
- Reads your current branch
- Finds the associated task
- Shows what to build
- Suggests next steps

**When to use:**
- Starting work on a task
- Resuming work after pause
- Need smart guidance

**Old way:** `/ccpm:implementation:start ISSUE-123` (needs issue ID)

---

### 3. Save Progress: `/ccpm:sync`

```bash
# In your feature branch: feature/PSN-30-auth
/ccpm:sync "Implemented JWT endpoints"
```

**What it does:**
- Finds your current task
- Saves progress to Linear
- Shows what you've changed
- Updates the task automatically

**When to use:**
- After completing a subtask
- Want to track progress
- Need to document what you did

**Old way:** `/ccpm:implementation:sync PSN-30 "message"` (needs issue ID)

---

### 4. Commit Changes: `/ccpm:commit`

```bash
# In your feature branch: feature/PSN-30-auth
/ccpm:commit "Add JWT token validation"
```

**What it does:**
- Creates a git commit automatically
- Formats it as conventional commit
- Links it to Linear
- Updates the task

**When to use:**
- After completing a working piece of code
- Ready to commit changes
- Want automatic formatting

**New feature!** Combines manual git commits + Linear comments

---

### 5. Verify Quality: `/ccpm:verify`

```bash
# In your feature branch: feature/PSN-30-auth
/ccpm:verify
```

**What it does:**
- Runs all quality checks
- Tests pass? âœ“
- Code style good? âœ“
- Performance ok? âœ“
- Verifies with you

**When to use:**
- Before creating PR
- Want to ensure quality
- Find issues early

**Old way:** `/ccpm:verification:check` + `/ccpm:verification:verify` (2 commands)

---

### 6. Complete Task: `/ccpm:done`

```bash
# In your feature branch: feature/PSN-30-auth
/ccpm:done
```

**What it does:**
- Final pre-flight checks
- Creates a pull request
- Updates Linear task
- Cleans up

**When to use:**
- Finished the feature
- Ready for review
- Want automatic PR creation

**Old way:** `/ccpm:complete:finalize PSN-30` (needs issue ID)

## Feature Flags

New features are safely behind feature flags:

**View status:**
```bash
/ccpm:config feature-flags --show
```

**Disable new commands (revert to old):**
```bash
/ccpm:config feature-flags optimized_workflow_commands false
```

**Enable new commands:**
```bash
/ccpm:config feature-flags optimized_workflow_commands true
```

## Troubleshooting

### "Can't find your issue. Please provide issue ID manually."

**Problem:** Branch name doesn't match standard format

**Solution:** Provide issue ID explicitly:
```bash
/ccpm:work PSN-30
/ccpm:sync "message" PSN-30
/ccpm:commit "message" PSN-30
```

**Prevent:** Use standard branch names:
```bash
git checkout -b feature/PSN-30-description
```

### "No commits to verify"

**Problem:** No git commits yet, can't verify

**Solution:** Make some commits first:
```bash
git commit -m "..."  # Make a commit
/ccpm:verify         # Then verify
```

### "Command failed with error: XYZ"

**Solution:** Check error message and run help:
```bash
/ccpm:help COMMAND   # Show command help
/ccpm:work --help    # Show work command help
```
