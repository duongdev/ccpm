### Q: How Do the Auto-Detect Commands Work?

A: New commands read your git branch name to find the issue ID automatically.

**Example:**
```bash
# Git branch: feature/PSN-30-add-auth
/ccpm:work    # ‚úì Auto-detects PSN-30
/ccpm:sync    # ‚úì Auto-detects PSN-30
/ccpm:commit  # ‚úì Auto-detects PSN-30
/ccpm:verify  # ‚úì Auto-detects PSN-30
/ccpm:done    # ‚úì Auto-detects PSN-30
```

**How it works:**
1. Reads current git branch name
2. Extracts issue ID (e.g., PSN-30)
3. Uses it automatically
4. No need to provide it!

**Supported formats:**
- `feature/PSN-30-description`
- `fix/PSN-30-description`
- `docs/PSN-30-description`
- Any branch with `PSN-30` in it

---

### Q: What If My Branch Name Doesn't Have the Issue ID?

A: Provide it manually:

```bash
# Branch: feature/my-auth-feature (no issue ID)
/ccpm:work PSN-30              # Provide ID explicitly
/ccpm:sync "Done" PSN-30       # Provide ID explicitly
/ccpm:commit "message" PSN-30  # Provide ID explicitly
```

**Better:** Use standard branch names:
```bash
git checkout -b feature/PSN-30-add-auth  # Includes issue ID
/ccpm:work  # Now auto-detects!
```

### Q: How Does `/ccpm:commit` Work?

A: It automates git commits with Linear linking.

```bash
/ccpm:commit "Add JWT token validation"
```

**What it does:**
1. Creates a git commit
2. Auto-formats as conventional commit: `feat(PSN-30): Add JWT token validation`
3. Links to Linear automatically
4. Updates the task

**Benefits:**
- Consistent commit format
- Automatic Linear linking
- Follows project conventions
- 60% token reduction

**Works with:**
- Any branch with issue ID in name
- Manual issue ID if needed
- All project configurations

## Feature Flags Questions

### Q: What Are Feature Flags?

A: Controls that let you enable/disable new features safely.

**Purpose:**
- Gradual rollout (not everyone at once)
- Safe testing (can disable instantly)
- User choice (you control which features you use)
- A/B testing (compare old vs new)

**Example:**
```bash
# Disable new commands, use old ones
/ccpm:config feature-flags optimized_workflow_commands false

# Enable new commands
/ccpm:config feature-flags optimized_workflow_commands true
```

## Troubleshooting Questions

### Q: "Command Failed: Can't Find Your Issue"

A: Issue auto-detection failed.

**Causes:**
- Branch name doesn't contain issue ID
- Issue ID format not recognized
- Issue not yet created in Linear

**Solutions:**
1. Use standard branch format:
   ```bash
   git checkout -b feature/PSN-30-description
   ```

2. Or provide issue ID explicitly:
   ```bash
   /ccpm:work PSN-30
   ```

3. Or create issue first:
   ```bash
   /ccpm:plan "New feature" my-app
   ```

---

### Q: "Command Failed: No Commits to Verify"

A: You haven't made any git commits yet.

**Solution:**
```bash
git add .
git commit -m "initial work"
/ccpm:verify  # Now it works
```

---

### Q: "Command Failed: Feature Flag Disabled"

A: A required feature flag is disabled.

**Solution:**
```bash
/ccpm:config feature-flags --show  # See status
/ccpm:config feature-flags optimized_workflow_commands true  # Enable
```

---

### Q: "/ccpm:commit Not Creating Proper Format"

A: Commit message format issue.

**Ensure:**
1. Proper branch name with issue ID
2. Clear, concise message
3. Feature flag enabled

**Example that works:**
```bash
git checkout -b feature/PSN-30-auth
/ccpm:commit "Add JWT token validation"
# Creates: feat(PSN-30): Add JWT token validation
```

---

### Q: Auto-Detection Not Working

A: Branch name not matching expected format.

**Debug:**
```bash
git branch  # Shows current branch
/ccpm:work --debug  # Shows what it's trying to detect
```

**Solution:**
```bash
# Use standard format
git checkout -b feature/PSN-30-description

# Or provide issue ID
/ccpm:work PSN-30
```
